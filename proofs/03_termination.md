<h1 align="center">Proof of Termination</h1>

> The investigation will finish in finite time.

---

## Theorem

**HTSA terminates in finite time for any investigation graph G.**

---

## Setup

Given:
- G = (V, E, P, Ev) is a finite DAG with |V| = n nodes
- OPEN is a priority queue initialized with {v₀}
- Each iteration either expands, prunes, or classifies a node

---

## Proof

**Step 1: The graph is finite.**

V is a finite set with |V| = n. There are at most n nodes that can ever be explored.

**Step 2: The graph is acyclic.**

G is a DAG — no directed cycles exist. Therefore, no node can be reached from itself via any sequence of edges. This means the algorithm cannot loop back to a previously visited node through expansion.

**Step 3: Each node is processed at most once.**

When a node v is extracted from OPEN (step 3 of the algorithm), it is removed from the queue. Each node can only be inserted into OPEN once — when it is generated as a child during EXPAND (step 4). Since G is acyclic, a node generated as a child of v cannot be v itself or any ancestor of v.

**Step 4: Each iteration reduces |OPEN| or produces a terminal outcome.**

For each node v extracted from OPEN, and for each child c generated by EXPAND(v), exactly one of three things happens:

```
(a) c fails the counterfactual test → DISCARDED     (never enters OPEN)
(b) P(c) < θ_prune                → PRUNED          (never enters OPEN)
(c) DEPTH_CRITERIA(c) = true      → added to R      (never enters OPEN)
(d) DEPTH_CRITERIA(c) = false     → added to OPEN   (will be processed later)
```

In case (d), c is added to OPEN — but c has strictly greater depth than v. Since the maximum depth of any node in a finite DAG is at most n - 1, this can happen at most n - 1 times along any path.

**Step 5: Termination follows.**

- OPEN starts with 1 element
- Each extraction removes 1 element
- Each expansion adds at most b children (branching factor) to OPEN
- But the total number of nodes that can ever be added is bounded by |V| = n
- Therefore, after at most n extractions and expansions, OPEN is empty
- The WHILE loop condition (OPEN ≠ ∅) becomes false
- The algorithm returns R and halts

---

## Formal Statement

```
Let T(n) = total number of iterations.

T(n) ≤ n

where n = |V|, the total number of nodes in G.

Since n is finite, T(n) is finite. ∎
```

---

## Bound

The algorithm performs at most **n node extractions**, where n is the total number of nodes in the investigation graph. Each extraction involves constant-time operations (counterfactual test, Bayesian update, depth criteria check) plus the cost of evidence gathering (which is external to the algorithm).

The computational complexity of the algorithm itself (excluding evidence gathering) is:

```
O(n log n)

where the log n factor comes from priority queue operations.
```
